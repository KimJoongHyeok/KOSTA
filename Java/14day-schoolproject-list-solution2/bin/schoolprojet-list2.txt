리팩토링
리팩터링은 소프트웨어 공학에서 '결과의 변경 없이 코드의 구조를 재조정함'을 뜻한다.
주로 가독성을 높이고 유지보수를 편하게 한다.
버그를 없애거나 새로운 기능을 추가하는 행위는 아니다.

SchoolProject List Version 을 refactoring 해본다

Test class 오 실행 결과는 기존과 동일
business logic 을 정의한 SchoolService가 리팩토링 대상

	기존			
	add()	 -> 중복확인을 위한 for문을 이용, list의 요소 탐색		
	find()	 -> 검색을 위해 for문을 이용, list 요소 탑색
	delete() -> 삭제를 위해 for문을 이용, list요소 탐색		

	변경
	공통적으로 적용되는 for문을 이용해 list요소 탐색 기능을 별도의 메서드에서 정의 ->findIndexByTel(tel) : int
	add(),find().delete()에서는 위 기능을 재사용
	
	


- 요구사항
	학교 구성원 정보를 관리하는 시스템을 구축하고자 한다
	학교 구성원은 학생,교사,직원으로 구성된다
	
	학생은 전화번호, 이름, 주소, 학번으로 구성된다  
	교사는 전화번호, 이름, 주소, 과목으로 구성된다
	직원은 전화번호, 이름, 주소, 부서로 구성된다
	
- 주요 기능
	1. 구성원 정보를 등록
		학교 수성원 정보(학생,교사,직원)를 등록할 수 있다.
		등록시 기존 구성원 전화번호와 동일한 번호가 있을 경우 등록시키지 않는다.
		( 중복시 메시지 : 01012341234 tel이 중복되어므로 등록 불가합니다.)
	
	2. 전체 구성원 리스트 출력( 상세정보가 모두 출력 )
		
	3. 전화번호를 이용한 구성원 정보 검색
		전화번호를 전달하면 그에 해당하는 구성원 정보를 반환
		만약 전화번호에 해당하는 구성원 정보가 없다면 null을 반환
	
	4. 전화번호를 이용한 구성원 정보 삭제
		전화번호를 전달하면 구성원 정보를 삭제한다
		만약 전화번호에 해당하는 구성원 정보가 없어서 삭제할 수 없다면 아래와 같이 메시지를 제공한다.
		( 01012341234 tel에 해당하는 구성원 정보가 없어서 삭제 불가합니다. )
